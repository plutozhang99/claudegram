## Key Decisions & Accepted Risks

### Architecture (P2 — locked from architect review pass 2026-04-18 + user Q1–Q5 confirmation)
- **Two-registry split**: `InMemoryHub` (PWA fan-out) + `SessionRegistry` (fakechat targeted send). Disjoint keys/failure modes.
- **SessionRegistry eviction-on-rebind**: re-`register` evicts + closes prior socket. R2 hardened with try/catch + socket-aware Disposable.
- **Reply failure mode = error frame + drop** (architect option a). `client_msg_id` echoed for correlation.
- **`/session-socket` auth gate at upgrade time**: pre-upgrade 401 when `TRUST_CF_ACCESS=true` + missing/empty headers. Shape check only; CF edge does crypto in P4.
- **`mark_read` semantics**: ts-based monotonic max with `MAX(COALESCE(last_read_at,0), ?)`. Cross-session validation via composite SQL key + defensive runtime check.
- **Echo dedup = origin tag (Q1=a)**: claudegram tags forwarded replies `origin:'pwa'`; fakechat skips outbound `/ingest`.
- **Retry queue cap = 100** (Q2); **bufferedAmount cap = 1 MB** (Q3); **Tech-debt bundle in P2** (Q4); **Chaos → P3** (Q5).
- **Reconnect jitter**: ±20% on fakechat exp backoff (250ms → 8000ms).
- **Inbound validation policy**: error frame on first malformed; close 1003 at N consecutive (default 5).
- **Phase merge**: P2.4 (WS client) + retry queue merged.
- **No schema change in P2**: `last_read_at` already exists.
- **Route discriminant via `ws.data.kind`**: `'user-socket' | 'session-socket'`. Bun.serve typed with WsData union. Closes the P4 auth-class privilege-escalation risk.
- **Exhaustive switch on `SessionRegistry.SendResult.reason`** in user-socket.ts with `never`-typed default — **P2.5's `'buffer_full'` arm WILL produce a compile error there until handled. This is by design.**
- **Shared `WsErrorReason` union** in `_ws-helpers.ts`: `'invalid_payload' | 'session_not_connected' | 'unknown_message' | 'internal_error' | 'send_failed'`.
- **`InboundReply` (fakechat) ↔ `OutboundSessionPayload` (claudegram)**: structurally identical; defended by a compile-time identity check inside `claudegram-client.ts` (a shared-types package would require monorepo setup; deferred).
